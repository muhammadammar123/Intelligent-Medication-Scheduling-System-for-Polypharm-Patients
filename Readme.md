# Intelligent Medication Scheduler 

## Problem
Polypharmacy patients (10+ medications) need daily schedules satisfying:
- **Dosing intervals** (e.g., every 8 hours between same-drug doses)
- **Drug interactions** (e.g., Metformin and Omeprazole ≥1 hour apart)
- **Food-timing rules** (with meal, empty stomach, before/after meal)
- **Soft preference**: avoid nighttime doses (00:00–06:30, 21:00–23:30)

## Algorithm
Informed Backtracking Search (CSP, AIMA Chapter 6) with:
- **MRV** — Minimum Remaining Values variable ordering
- **LCV** — Least Constraining Value ordering
- **Forward Checking** — in-place domain pruning with undo stack
- **A\* Heuristic** — admissible h(n) = Σ min slot costs over unassigned variables

## Technical Overview

### CSP Formulation
| Component   | Encoding                                              |
|-------------|-------------------------------------------------------|
| Variables   | (med_id, dose_index) — one per individual dose        |
| Domains     | Subsets of {0..47} filtered by food constraints       |
| Constraints | Dosing intervals + drug interactions + food rules     |

### Heuristic (Admissible)
```
h(n) = Σ  min  C(s)
       v∈U  s∈D(v)
```
where U = unassigned variables, D(v) = pruned domain, C(s) = night penalty.
Admissible because it solves a relaxed problem (constraints removed between variables).

### Search Flow
1. MRV selects the most constrained unassigned dose variable
2. For each candidate slot, tentatively assign → forward check → compute f = g + h
3. Try slots in ascending f order (A\*-style informed ordering)
4. Forward checking prunes neighbours in-place; undo stack restores on backtrack
5. First complete assignment found is returned (optimal under soft cost)
